# Roadmap: Ближайший Релиз

> Обновлено: 2026-03-28
> Текущая версия: 1.4.4-dev
> Целевой ближайший релиз: 1.4.5

## Контекст

Проект работает на Windows 10/11 и использует `windows-capture`.
Фокус ближайшего релиза — стабилизация после добавления вкладки API,
улучшение качества и подготовка к `v1.5.0`.

## Цели `v1.4.5`

1. Закрыть краткосрочные регрессии и UX-недочёты в API-вкладке GUI.
2. Выровнять quality gates (lint/type/tests) для измененных модулей.
3. Подготовить минимальный мост к `v1.5.0` (реальный transport для событий
   как отдельная задача следующего этапа).

## Scope `v1.4.5` (краткосрочный)

### P0

1. Длинный smoke-run API вкладки (30+ минут) без деградации лог-окна.
2. Проверка edge-cases live-логов API (ротация, пустой файл, быстрый
   перезапуск).

### P1

1. UX-доработки `recent recordings` (фильтр + быстрые действия).
2. Закрепление quality gates для затронутых модулей.
3. Подготовка технического дизайна для WebSocket transport (`v1.5.0`).

### P2

1. Полировка UX для API/GUI по итогам smoke-run.
2. Подготовка задач декомпозиции `v1.5.0`.

## За рамками `v1.4.5`

1. Полноценный WebSocket transport.
2. Профили записи.
3. Post-recording flow.
4. Библиотека записей.
5. Service mode.

## Release Gates `v1.4.5`

1. `uv run ruff check .`
2. `uv run ruff format --check .`
3. `uv run pytest tests/unit/`
4. `uv run pytest tests/integration/` (допускается targeted-run при
   ограничениях окружения)
5. Нет открытых P0 дефектов в критичном пути GUI/API.
6. Обновлены `CHANGELOG.md` и `TODO.md`.
