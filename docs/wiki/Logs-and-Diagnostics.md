# Logs and Diagnostics

## Расположение логов

- Общие логи приложения: `logs/mia_YYYY-MM-DD.log`.
- Логи API: `logs/api/api_YYYY-MM-DD.log`.

Логи API хранятся в отдельной папке по дням.

## Что пишется в API-логи

- Запуск и остановка API сервера.
- Access-логи HTTP-запросов:
  - метод
  - путь
  - HTTP статус
  - latency
  - `request_id`
  - IP клиента
- Ошибки в обработчиках API.

Пример строки:

```text
API POST /api/v1/start -> 200 (657.00 ms) request_id=... ip=127.0.0.1
```

## Логи в реальном времени в GUI

На вкладке `API` отображается текущее содержимое файла API-лога:

- автообновление каждую секунду;
- чтение файла выполняется в фоне и не блокирует UI;
- кнопка ручного обновления;
- автопрокрутка к последним строкам;
- переключение на новый дневной файл автоматически.

## Открытие папки логов

- Через вкладку `API` кнопкой `Открыть папку логов API`.
- Через код используется функция `open_api_logs_folder()`.

## Диагностика

Для быстрой проверки используйте вкладку `Диагностика` в GUI.

- Вкладка диагностики использует тот же readiness checklist, что и
  compact readiness center на вкладке `Запись`.
- Для частых проблем доступны one-click действия: выбор папки вывода,
  обновление окон/микрофонов и переход к инструкции по FFmpeg.

Полезные команды:

```powershell
Invoke-WebRequest http://127.0.0.1:5000/health -UseBasicParsing
ffmpeg -version
```
