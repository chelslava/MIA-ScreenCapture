# Регрессионный Сценарий: GUI API Tab (`v1.4.5`)

> Обновлено: 2026-03-29
> Цель: стабильная проверка `start/stop/restart` при открытом live-логе.

## Предусловия

1. Запуск приложения: `uv run python main.py --gui`.
2. Открыта вкладка API.
3. Включён API сервер и отображается порт.
4. Окно логов API открыто и активно обновляется.

## Сценарий A: Start/Status/Stop

1. Нажать `Запустить`.
2. Выполнить `GET /health` и `GET /api/v1/status`.
3. Выполнить `POST /api/v1/start` с валидным payload.
4. Убедиться, что в live-логе есть запись запроса `/api/v1/start` со
   статусом `200`.
5. Выполнить `POST /api/v1/stop`.
6. Убедиться, что в live-логе есть запись `/api/v1/stop` со статусом `200`.

Критерий успеха:
- Нет `TimeoutError` в GUI потоке.
- UI не зависает во время запросов.

## Сценарий B: Restart API сервера

1. Нажать `Перезапустить`.
2. Дождаться статуса, что сервер снова запущен.
3. Повторить `GET /health`.
4. Проверить, что порт и токен остались прежними.

Критерий успеха:
- После перезапуска API доступен без ручного повторного запуска GUI.
- Live-лог продолжает обновляться.

## Сценарий C: Нагрузка на live-лог

1. Выполнить серию из 30-50 запросов `GET /api/v1/status`.
2. Во время серии прокручивать окно логов.
3. Проверить, что интерфейс остаётся отзывчивым.

Критерий успеха:
- Нет заметной деградации UI.
- Логи не "зависают" и продолжают дополняться.

## Артефакты проверки

1. Вставить результат в `plans/release-note-v1.4.5-smoke.md`:
   - дата/время прогона;
   - порт API;
   - итог по сценариям A/B/C;
   - найденные дефекты (если есть).
